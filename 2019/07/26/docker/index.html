concept of docker

The concetp

A plateform achieves a struture between apps and os system below as a system layer. Docker offers all everything needed to run the apps above, including code, system libraries, settings.

Why does the system need such a layer?
Reason: different type of os sytem: linux,windows and so on. Above os system layer, all system libraries are modified to fit the system type, apps reply on differnent system libraries, which easily causes bugs. To avoid bugs, a system-crossing plateform is needs. As a result, docker as a container comes out.

The function of container is similar with a Virtual Machine. Two things to clearify:

  1. What is Virtual Machine ?
    The storage is cut out a static part as a individual environment. This part initial a environment to offer all requirements for apps. By this method, the computer has two kinds of os system.
  2. The differnece between docker and Virtual Machine.
    A container is at a higher level than the method of Virtual Machine.
