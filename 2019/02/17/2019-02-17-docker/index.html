为什么要docker

docker做到了什么，能够让我来引进docker。虽然早闻docker大名，也大概知道docker的概念，但直到今天帮助同学做笔试project时，才想起来要入个门。
那么，docker的引入给我们带来了什么呢？docker的结构是什么样的？根据看概念，我理解docker为一个容器。这个容器可以根据使用者的需求来配置相应的环境。
本质上，这仍然是一个独立的环境。与虚拟机的区别又是什么？
抛出若干问题。
1.环境的配置问题。环境配置的可变要素是安装包与环境，日常开发中，安装包基本都是最纯净的安装包，但环境各有不同。有开发环境，生产环境，测试环境等。
如果将环境要素统一呢？例如，将发布环境打包起来，在测试环境和开发环境模拟配置相同环境。经过上述简化操作，环境配置的冲突就不会产生了。
因此，docker的思路是将环境作为一个镜像，作为共有资源，开发者都能获取这个资源。
提问，环境资源是深拷贝吗？还是公用的？个人认为是深拷贝。如果是公用的话，结构更倾向于云服务器，那不是将三个环境统一，而是直接将三个环境合并。
2.应用间隔离。
提问，LXC(Linux Containers)作为linux的容器，是均分了内核资源了吗？
3.mac系统和windows系统兼容。
windows和mac本质上都是unix的内核。那么，这样理解的话，docker是抽取出其中相同的部分建立的内容？

docker与虚拟机的区别

和我理解的一样。虽然上层环境不同，但是所有的结构都使用相同的上层环境。

docker的简单入门

docker的关键概念，是image，container，repository。
image理解为镜像，主要作用是提供环境。只读。
container理解为沙盒，是镜像的实例。
repository镜像仓库。
如果以linux为例，image理解为linux的镜像文件，container为实际的linux实例，repository则是镜像文件仓库。
提问，对于一个复杂的环境来说，一定是由多个镜像组合而成的。在这种情况下，repository和image区别是什么？应该是多个image被repository存放。
提问2，repository是一个完整的个体还是每个环节都有所不同？或者说，repository是作为单个用户的，但是存在于全局中。我认为是两者都存在。
